# 核心要求

## 1. 导航功能（必需）

- **圆点导航**：底部导航栏左侧显示圆点组，点击圆点可切换对应页面
- **左右箭头**：导航栏右侧显示左右箭头按钮，点击切换上一页/下一页
- **全屏功能**：导航栏包含全屏按钮，点击切换浏览器全屏模式
- **键盘快捷键**：
  - `←` / `→`：上一页/下一页
  - `Ctrl+F` / `F`：全屏切换

## 2. 编辑模式（必需）

- 导航栏包含编辑模式切换按钮
- 点击后所有文本元素变为可编辑状态（`contenteditable="true"`）
- 可编辑元素 hover 时显示视觉提示（如虚线边框）
- 再次点击退出编辑模式，保留修改内容

## 3. 美观设计

- 整体风格时尚大方，具有现代感
- 色调前后一致，基调统一，避免跳跃式配色变化
- 合理运用留白、阴影和过渡效果提升视觉品质

## 4. 技术实现

- **单文件 HTML**：所有 CSS 写在 `<style>` 标签内，所有 JavaScript 写在 `<script>` 标签内
- **不依赖外部库**：不使用 React、Vue、jQuery 等外部依赖
- 代码结构清晰，功能完整可用

# 输出要求

- 最终文件必须是有效的 HTML5 文档
- 所有功能必须实际可用（圆点导航、箭头切换、全屏、编辑模式）
- 代码必须完整，不要使用 `/* ... */` 省略
- 直接输出最终文件内容并保存到目标路径，不要输出与任务无关的闲聊
