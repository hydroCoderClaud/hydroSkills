# Code Review

对指定的代码文件或代码片段进行全面审查，从以下维度分析并给出建议：

## 审查维度

1. **代码质量** - 命名规范、代码风格、可读性
2. **逻辑正确性** - 边界条件、异常处理、逻辑漏洞
3. **性能** - 不必要的计算、内存泄漏、算法效率
4. **安全性** - XSS、SQL注入、敏感信息泄露等 OWASP Top 10
5. **可维护性** - 代码复杂度、耦合度、重复代码

## 输出格式

对每个发现的问题，按以下格式输出：

- **严重程度**: 🔴 高 / 🟡 中 / 🟢 低
- **位置**: 文件名:行号
- **问题**: 简述问题
- **建议**: 给出修复方案

最后给出总体评分（1-10）和总结。
